# SearXNG Configuration

SearXNG can be used to provide web search capabilities for Swarpius. Here we discuss local SearXNG configuration for the Docker Compose stack.

[SearXNG](https://github.com/searxng/searxng) is licensed under [AGPL-3.0](https://www.gnu.org/licenses/agpl-3.0.html). The image is pulled directly from Docker Hub when you opt in to the `search` compose profile; Swarpius itself does not redistribute it. See the root README's [Web search](../README.md#web-search-strongly-recommended) section for the licensing context.

## What lives here

- `entrypoint.sh`: small wrapper around the SearXNG image's own entrypoint. On first boot it copies the image's bundled `settings.yml` into the config directory; on every boot it idempotently patches the `search:` block to ensure `formats: [html, json]` is set. The agent's `web_search` tool needs the JSON path; the image's default is HTML only. We use this wrapper so we don't have to redistribute a modified copy of the original (AGPL-3.0) `settings.yml`.

Both `/etc/searxng/` (config) and `/var/cache/searxng/` (cache) are stored in named Docker volumes — `searxng-config` and `searxng-cache` respectively. The container manages them; nothing surfaces on the host filesystem. Swarpius's use of SearXNG is narrow (web search backend for the agent's `web_search` tool only), so the convenience of host-side editing isn't needed.

### Inspecting / customising the running settings

If you ever need to look at or tweak the live `settings.yml`, exec into the container:

```bash
docker compose exec searxng sh -c 'cat /etc/searxng/settings.yml'
docker compose exec searxng sh -c 'sed -i "..." /etc/searxng/settings.yml'    # edit
docker compose restart searxng
```

For more substantial customisation, the recommended path is to run your own SearXNG instance and point Swarpius at it via `SEARXNG_URL` in `agent/.env` — Swarpius is just a SearXNG client.

## Docker

From the monorepo root:

```bash
docker compose --profile search up -d
```

SearXNG is opt-in — it runs only when the `search` profile (or `all`) is active. Once running, set `WEB_SEARCH_PROVIDER=searxng` in `agent/.env` and restart the agent so the `web_search` skill is registered.

| Setting | Default | Override variable |
|---|---|---|
| Container port | 8080 | -- |
| Host port | 8888 | `SEARXNG_PORT` |
| Bind address | 127.0.0.1 (loopback only) | `SEARXNG_BIND_IP` |
| Session secret | random per host (generated on first boot) | `SEARXNG_SECRET` *(optional override)* |

The session secret is generated by SearXNG's own entrypoint the first time `settings.yml` is created and persists in the file thereafter, so each host gets its own. Override with `SEARXNG_SECRET` (e.g. `openssl rand -hex 32`) if you'd rather manage it explicitly.

## Connecting to the agent

Set in `agent/.env`:

```
SEARXNG_URL="http://searxng:8080"       # inside Docker network
SEARXNG_URL="http://localhost:8888"     # from host machine
```
